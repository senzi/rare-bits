# Rare Bits 🏛️

> “在无穷的熵增中，秩序是万中无一的巧合。”

本仓库是一个专门用于记录、搜集并存证那些在数学、统计学或密码学意义上极低概率出现的“数字异常”与“稀有比特”的档案馆。

## 🌌 仓库宗旨
在数字与概率的荒原里，纯粹的秩序是极其罕见的。当我们捕捉到一个违背随机混沌规律的实体时——无论它是一段特殊的字符串、一张具有数学巧合的图像，还是一个罕见的二进制序列——它都是值得被永恒记录的“数字宝石”。

**Rare Bits** 利用 Git 提交记录（Commit History）的不可篡改性，为这些发现提供原始的**发现证明（Proof of Discovery）**和永久的展示橱窗。

---

## 🛠️ “发现证明” 存证流程
为了确保每一个奇迹时刻的真实性，每一个新条目都遵循以下工作流：

### 第一步：创建独立实验目录
在仓库根目录下为每一个发现创建一个唯一的命名目录。该目录是一个完整的、自包含的“小试验”。
* **内容**: 包含说明文档（`.md`）、原始数据、二进制文件、搜索脚本或验证程序等。
* **操作**: `git add [目录名]` 并执行 `git commit -S -m "archive: discovery [发现代号]"`。
* **意义**: Commit 哈希值和服务端的时间戳构成了该发现最原始的物理证明。

### 第二步：元数据关联与回溯
在完成初始推送（Push）后，获取该 Commit 的哈希值，并在该目录下的说明文件中记录：
* **存证 Hash**: 关联到首次引入该发现的特定提交链接。
* **稀有度说明**: 描述该实体在数学或统计学上的稀有程度。无需提前解释产生细节，仅标注其稀缺性。

---

## 📂 扁平化存储结构

仓库采用根目录下“一发现一目录”的模式，保持每个条目的独立性与自由度：

`/[YYYY-MM-DD-发现代号]/`

### 目录内自由结构示例：
- `README.md`: 发现说明、稀有度分析及存证哈希。
- `data.*`: 原始数据文件（文本、十六进制等）。
- `image.* / video.*`: 视觉化证明或展示。
- `search_script.py`: 寻找该异常的原始脚本。
- `verify.sh`: 用于验证属性的快捷工具。
